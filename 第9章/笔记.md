9.1 实际用户 ID 和实际组 ID
• 实际用户 ID (UID) 和 实际组 ID (GID) 代表了进程的“真实身份”，也就是启动这个进程的用户和
组。
• 这两个 ID 通常在用户登录时由登录程序设置，并在整个会话期间保持不变。
• 可以通过 getuid() 和 getgid() 函数获取当前进程的实际 UID 和 GID。
9.2 有效用户 ID 和有效组 ID
• 有效用户 ID (EUID) 和 有效组 ID (EGID) 决定了进程在访问文件、执行操作时的权限。
• 默认情况下，有效 ID 与实际 ID 相同。但当程序设置了 Set-User-ID (SUID) 或 Set-Group-ID
(SGID) 位时，有效 ID 会被替换为程序所有者的 UID 或 GID。
• 可以通过 geteuid() 和 getegid() 函数获取当前进程的有效 UID 和 GID。
9.3 Set-User-ID 和 Set-Group-ID 程序
• Set-User-ID (SUID) 和 Set-Group-ID (SGID) 是可执行文件的特殊权限位。
• 当一个程序被设置了 SUID 位，其他用户执行它时，进程的有效 UID 会变成该程序所有者的 UID。
• 同理，SGID 位会让进程的有效 GID 变成程序所属组的 GID。
• 这是实现特权操作的常用方式，例如 passwd 命令就是一个典型的 SUID 程序，它允许普通用户
修改自己的密码（需要写入 /etc/shadow ）。
• 可以通过 chmod u+s <program> 设置 SUID， chmod g+s <program> 设置 SGID。
9.4 保存 set-user-ID 和保存 set-group-ID
• 保存的 set-user-ID (saved set-user-ID) 和保存的 set-group-ID (saved set-group-ID) 是进程的额
外凭证。
• 当一个 SUID/SGID 程序启动时，内核会将有效 ID 的值复制到保存的 ID 中。
• 这个保存的 ID 允许程序在需要时，临时放弃特权后再重新恢复。
• 可以通过 setuid() 、 seteuid() 等函数在实际 ID、有效 ID 和保存的 ID 之间切换。
9.5 文件系统用户 ID 和组 ID
• 文件系统用户 ID (fsuid) 和 文件系统组 ID (fsgid) 是 Linux 特有的概念，专门用于文件系统的权限
检查。
• 大多数情况下，fsuid 等于有效 UID，fsgid 等于有效 GID。
• 只有在非常特殊的场景下（如 NFS 服务器），才需要通过 setfsuid() 和 setfsgid() 来
显式修改它们。
• 普通程序一般不需要关注这两个 ID。
9.6 辅助组 ID
• 除了主组（由 GID 指定），一个用户还可以属于多个辅助组。
• 进程的辅助组 ID 列表继承自启动它的用户，这些组会在文件权限检查时被用到。
• 可以通过 getgroups() 获取当前进程的所有辅助组 ID。
