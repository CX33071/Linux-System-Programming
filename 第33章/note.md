1.线程栈
线程栈：每个线程都有自己独立的栈空间，用于存储局部变量、函数调用帧等。
默认大小：Linux 上默认线程栈大小通常为 8MB（可通过ulimit -s查看）。
栈溢出：如果线程栈空间耗尽，会触发SIGSEGV信号，导致整个进程崩溃。
自定义栈大小：在创建线程时，可以通过pthread_attr_t属性对象设置栈大小
2.信号和线程
信号归属：信号是发给进程的，所以进程内的任意线程都可以接受和处理它
信号处置：信号处理函数是进程级的，所有线程共享同一套信号处置方式
信号掩码：每个线程有自己独立的信号掩码，用于阻塞特定信号
int pthread_sigmask(int hoe,const sigset_t*set,sigset_t*oldset)用于修改当前线程的信号掩码
SIG_BLOCK将set中的信号添加到当前掩码中
SIG_UNBLOCK将set中信号从当前掩码中移除
SIG_SETMASK将当前掩码设置为set
int pthread_kill(pthread_t thread,int sig)向指定线程发送信号
发送SIGKILL或SIGSTOP到线程会直接终止或停止整个进程
- 处理异步信号
    在主线程中阻塞所有信号，创建所有子线程，子线程会继承主线程的掩码。
    创建一个专门的 “信号处理线程”，在该线程中解除对感兴趣信号的阻塞，并调用sigwait()或sigwaitinfo()来同步处理信号。
    避免在信号处理函数中做复杂操作，只设置标志位，由主线程或工作线程处理。
3.当多线程进程调用fork()时，子进程只继承调用fork()的那个线程，子进程会继承父进程的互斥量状态，如果父进程在fork()前持有锁，子进程中这些锁将永远处于锁定状态，导致死锁
4.用户级线程（ULT）：线程管理完全在用户态库中进行，内核对此一无所知。优点是上下文切换快，缺点是无法利用多核 CPU，一个线程阻塞会导致整个进程阻塞。
内核级线程（KLT）：内核直接管理线程，每个线程都是一个独立的调度实体。优点是可以并行在多核上运行，缺点是线程创建和切换开销较大。
现在用的linuxAPI都是内核级线程
5. - LinuxThreads 早期 Linux 的线程库 将每个线程实现为一个独立的进程  不符合 POSIX 标准
    - NPTL  每个线程有独立的 PID，但共享线程组 ID（TGID，即进程 ID）    支持真正的多核并行
6. Pthread API 的高级特性
    线程亲和力（Affinity）：pthread_setaffinity_np()，可以将线程绑定到特定的 CPU 核心上，避免缓存失效，提高性能。
    线程调度：pthread_setschedparam()，可以设置线程的调度策略和优先级（如 FIFO、RR）。
